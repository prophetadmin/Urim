# Repository History

This document records how the current URIM repository at HEAD was formed.

URIM has one active product identity at HEAD, but the repository preserves history from two sequential development lines that now live together in a single canonical repository.

## Lineage Summary

The repository history includes:

- an earlier URIM execution line
- a later runtime-completion continuation line that inherited earlier repository-owned URIM implementation artifacts and extended them further

These are not two active products, two competing roadmaps, or two competing repositories. They are two stages in the development history of the same URIM lineage.

## Earlier URIM Line

The earlier line reflects execution under [MetaDictum](https://github.com/prophetadmin/MetaDictum) Framework v1.

Its importance is historical and foundational: it established the earlier repository-owned URIM direction, artifacts, and execution record that the later continuation line inherited rather than replaced.

## Runtime-Completion Continuation Line

The later runtime-completion line was a continuation of URIM work, not a separate product.

It inherited prior URIM EXODUS material, pushed the runtime further into a runnable local application path, exposed live operational gaps, and generated the execution record from which later hardening observations were drawn.

That continuation line was not itself a MetaDictum v2-governed run. Some observations from it later informed what became MetaDictum v2, but that downstream framework history is not the governing subject of this repository.

## What Was Preserved

This repository preserves, in one place:

- the URIM product lineage
- the implementation continuation across both historical lines
- the governance and execution artifacts that define current authority at HEAD
- the historical provenance needed to understand how the present repository state was reached

Preservation here does not mean dual authority. History is plural, but active authority at HEAD is singular.
